### ~~P4 — IOTA Transaction Signing~~ ✅ DONE

Implemented `BuildTransaction`, `SignTransaction`, and `SubmitTransaction` for IOTA Rebased:

- **`BuildTransaction`**: Uses `unsafe_payIota` JSON-RPC to have the IOTA node construct BCS-serialized PTB transaction bytes. First queries `iotax_getCoins` to get available coin objects, then builds the payment transaction server-side.
- **`SignTransaction`**: Local Ed25519 signing following the IOTA Rebased protocol — constructs intent message (`[0x00, 0x00, 0x00] || tx_bytes`), Blake2b-256 hashes it, signs with Ed25519, produces IOTA signature format (`flag(1) || sig(64) || pubkey(32)`).
- **`SubmitTransaction`**: Calls `iota_executeTransactionBlock` JSON-RPC with base64-encoded tx bytes and signature.
- Added `TransactionBuilder` interface for the new RPC methods (`GetCoins`, `PayIota`, `ExecuteTransactionBlock`).
- Updated `UnsignedTransaction` and `SignedTransaction` structs with `TxBytes` field.
- Full test suite: mock-based unit tests, JSON-RPC httptest integration tests, end-to-end Build→Sign→Submit flow test.

**Files:** `internal/payments/iota_client.go`, `internal/payments/iota_client_test.go`

### ~~P7 — Rate Limiter Redis Backing~~ ✅ DONE

Refactored the setup wizard rate limiter from a hardcoded in-memory map to a `RateLimiter` interface with two implementations:

- **`memoryRateLimiter`**: In-memory sliding window (default, suitable for single-use wizard).
- **`redisRateLimiter`**: Redis Sorted Set (ZSET) backed sliding window. Survives process restarts, works across multiple wizard instances. Uses pipelined `ZREMRANGEBYSCORE` + `ZADD` + `ZCARD` + `EXPIRE` for atomic rate checking.
- Falls open on Redis errors for wizard usability.
- `NewRedisRateLimiter(nil)` gracefully falls back to in-memory.
- Removed duplicated rate limiting code from `HandleTestEmail` and `wizard_test_connections.go` — all handlers now use the shared `RateLimiter` interface.

**Files:** `internal/setup/ratelimit.go`, `internal/setup/ratelimit_test.go`, `internal/setup/wizard.go`, `internal/setup/wizard_test_connections.go`
