## Deferred Tasks

### P1 — Fix `TestUnit_Follow_SuccessWithPDS` (Pre-existing test failure)

**Package:** `internal/httpapi/handlers/social`
**Issue:** Missing `createSession` endpoint in `newFakePDS()` test helper.
**Fix:** Add `/xrpc/com.atproto.server.createSession` handler returning mock JWT tokens.
**Files:** `internal/httpapi/handlers/social/social_handlers_unit_test.go`

### P2 — Fix COMPOSE_PROFILES Mail Profile Bug

**File:** `internal/setup/writer.go:191`
**Issue:** Checks `SMTPHost == "mailpit"` but Docker email mode sets `SMTPHost = "localhost"`.
**Fix:** Check email mode (docker vs external) rather than SMTP host string.
**Also:** Always write `COMPOSE_PROFILES=` (even when empty) to prevent stale values; add `letsencrypt` profile when HTTPS with Let's Encrypt is configured.
**Files:** `internal/setup/writer.go`, `internal/setup/writer_test.go`

### P3 — Fix Redis AUTH Bug in Setup Wizard

**File:** `internal/setup/wizard_test_connections.go`
**Issue:** `HandleTestRedis` parses `password` field but never sends AUTH command before PING.
**Fix:** Use `go-redis` client library or send `AUTH <password>\r\nPING\r\n` before reading PONG.
**Files:** `internal/setup/wizard_test_connections.go`, `internal/setup/wizard_test_connections_test.go`

### P4 — IOTA Transaction Signing (Phase 2 Feature)

**Package:** `internal/payments`
**Issue:** `BuildTransaction`, `SignTransaction`, `SubmitTransaction` return `ErrNotImplemented`.
**Scope:**
- `BuildTransaction`: Construct Programmable Transaction Block (PTB) per IOTA Rebased spec
- `SignTransaction`: Ed25519 signing of PTB serialization
- `SubmitTransaction`: Call `iota_executeTransactionBlock` JSON-RPC
**Reference:** https://docs.iota.org/developer/iota-101/transactions/ptb/
**Files:** `internal/payments/iota_client.go`, `internal/payments/iota_client_test.go`

### P5 — Custom Platform Token (Phase 3 Feature)

**Issue:** No implementation exists for website-specific token on IOTA Rebased.
**Scope:**
- Deploy a Move module on IOTA Rebased testnet
- Token minting/transfer logic
- Integration with payment service
- Token balance display in video platform API
**Reference:** https://docs.iota.org/about-iota/

### P6 — Health Check Queue Depth

**File:** `internal/httpapi/health.go`
**Issue:** Hardcoded queue depth values (5, 10) instead of actual worker queue metrics.
**Fix:** Replace with real queue metrics from Redis or database.

### P7 — Rate Limiter Redis Backing

**Issue:** Setup wizard rate limiter uses in-memory `sync.Map`; resets on process restart.
**Fix:** Back with Redis for persistence. Low priority — wizard is single-use per deployment.

### P8 — `GetNodeStatus()` Test Coverage

**Package:** `internal/payments`
**Issue:** 0% test coverage for `IOTAClient.GetNodeStatus()`.
**Fix:** Add unit tests covering success and error paths.
